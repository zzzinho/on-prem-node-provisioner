1) Label your nodes so each ONP component lands where it must:

     # always-on nodes (control plane / never power off) host the controller + wol-agent
     kubectl label node <always-on-node> onp.io/always-on=true
     # wakeable nodes ONP powers on/off host the shutdown-agent
     kubectl label node <wakeable-node>  onp.io/managed=true

   Verify the workloads scheduled:

     kubectl -n {{ .Release.Namespace }} get deploy,ds,svc -l app.kubernetes.io/instance={{ .Release.Name }}

{{- if not .Values.namespace.create }}

   NOTE: the shutdown-agent is privileged. If this cluster enforces Pod Security
   Admission, label the namespace so it may schedule:

     kubectl label ns {{ .Release.Namespace }} pod-security.kubernetes.io/enforce=privileged
{{- end }}

2) Register a Machine for each WAKEABLE physical node. The Machine is cluster-
   scoped; capacity is the source of truth for fit checks while the node is off:

     apiVersion: onp.io/v1alpha1
     kind: Machine
     metadata:
       name: worker-1
     spec:
       nodeName: worker-1
       power:
         provider: wol
         wol:
           macAddress: "aa:bb:cc:dd:ee:ff"
           # broadcastAddress: 192.168.1.255   # optional; defaults to 255.255.255.255
       capacity:
         cpu: "8"
         memory: 32Gi

3) Trigger a wake (Phase 1 / M2 manual path) with the annotation:

     kubectl annotate machine worker-1 onp.io/wake-now=true

   Follow the lifecycle:

     kubectl get machines
     kubectl describe machine worker-1

Notes:
- The wol-agent runs hostNetwork on always-on nodes because Wake-on-LAN magic
  packets are L2 broadcasts and must originate on the target node's segment.
- The wol-agent holds NO Kubernetes API permissions by design.
